#2 Les frameworks

Les nouveaux besoins imposent des changements d'architecture :

  • Temps de réponse instantané
  • Gestion des réseaux lents, aléatoires...
  • Mode déconnecté

Les architectures SPA (Single Page Application)
sont une solution.

Architecture classique

Architecture "SPA"

AngularJS

  • Framework SPA par Google
  • Découplage "MVC"
  • Complètement testable
  • Double binding
  • Enorme communauté
  • Remplacé par Angular ("tout court")

Angular

  • Réécriture complète
  • Approche Components
  • Ecrit en TypeScript
  • Nouveau rythme de releases (2 / an)
Example

React

  • Basé sur les Components
  • Uniquement la partie vue
  • Plus de simplicité...
  • ... mais demande de créer son écosystème
Example

VueJS

  • Proche d'AngularJS (v1)
  • Suscite beaucoup d'intérêt
  • Très facile d'accès
  • Pas de grosses entreprise derrière
  • v3 prévue en 2019
